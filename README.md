# HAKATON

Автоматизированная система обработки корпоративной почты на Python. Приложение читает письма из папки `inbox`, определяет категорию по содержимому и раскладывает файлы по папкам внутри `processed`.

---

## Возможности

- классификация корпоративных писем по ключевым словам и весам
- обработка русскоязычных и частично англоязычных писем
- проверка обязательных полей отправителя и получателя
- отдельная обработка черновиков, ошибок формата и изображений
- ручная доразметка писем, которые не удалось уверенно распределить
- сохранение итогового отчёта в `processed/log.txt`

---

## Что делает программа

Программа открывает папку `inbox`, читает все файлы с письмами и проверяет каждое:

1. есть ли отправитель
2. есть ли получатель
3. какой текст написан в письме
4. есть ли в тексте слова, похожие на одну из известных категорий

После проверки программа выбирает подходящую категорию и переносит письмо в отдельную папку внутри `processed`.

```text
Письмо про оплату      ->  processed/Счета/
Письмо про отпуск      ->  processed/HR/
Письмо про сбой        ->  processed/Важное/
Письмо без получателя  ->  processed/Черновик/
Сломанный файл         ->  processed/Ошибки/
```

Если программа не уверена, куда отнести письмо, она помещает его в `Не распределено` и предлагает выбрать категорию вручную.

В конце работы программа показывает итог в терминале и создаёт файл `processed/log.txt`.

## Категории писем

| Категория | Что попадает |
|---|---|
| Важное | Срочные обращения, критические сбои, массовые инциденты |
| Технические ошибки | Проблемы с техникой, ПО, интернетом, сервисами и оборудованием |
| Подтверждение доступа к аккаунту | VPN, GitLab, Confluence, 1C, логины и восстановление аккаунтов |
| Информация | Организационные уведомления, отчёты, созвоны, дайджесты |
| Документы | Договоры, акты, согласования, инструкции |
| Счета | Счета, invoice, реквизиты, оплаты |
| HR | Отпуска, больничные, резюме, переводы и кадровые изменения |
| Спам | Фишинг, розыгрыши, подозрительные ссылки и пустые письма |
| Черновик | Письма без получателя |
| Ошибки | Письма без отправителя, неподдерживаемые форматы |
| Не распределено | Письма без уверенного совпадения с известными категориями |
| Картинки | Файлы `.jpg`, `.jpeg`, `.png` без связанного текстового письма |

## Как программа определяет категорию

Классификатор приводит текст к нижнему регистру, разбивает на токены, убирает знаки пунктуации и стоп-слова, затем применяет стемминг. После этого сравнивает слова с весовыми словарями категорий.

Например, если в письме встречаются слова `счет`, `оплата`, `invoice` — письмо попадает в `Счета`. Для каждого слова задан свой вес. Чем больше подходящих слов, тем увереннее выбирается категория.

Если максимальный вес ниже порога `THRESHOLD = 0.05`, письмо попадает в `Не распределено`.

## Нюансы кода

Основная логика находится в файле `distribution_of_email.py`.

- **EmailReader** — ищет в письме отправителя, получателя, тему и основной текст
- **EmailClassifier** — определяет категорию письма по словарю с весами
- **EmailProcessor** — проходит по всем файлам в `inbox`, проверяет формат и вызывает классификатор
- **EmailSorter** — создаёт папки внутри `processed` и переносит туда письма
- **interactive_sort** — запускает ручной выбор категории для писем из `Не распределено`

Важные особенности:

- программа работает с `.txt` и файлами без расширения
- файлы других форматов попадают в `Ошибки`
- изображения `.jpg`, `.jpeg`, `.png` обрабатываются отдельно; если у картинки есть текстовое письмо с таким же именем — картинка получает ту же категорию
- письма перемещаются из `inbox` в `processed`, а не копируются
- папка `processed` создаётся автоматически

## Установка и запуск

### Шаг 1. Скачайте проект

**Через ZIP:**

1. Откройте страницу проекта на GitHub
2. Нажмите зелёную кнопку `Code` → `Download ZIP`
3. Распакуйте архив в удобное место

**Через Git:**

```
git clone https://github.com/username/HAKATON.git
```

---

### Шаг 2. Установите зависимости

Для работы программы нужна библиотека `nltk`.

Сначала создайте виртуальное окружение в папке проекта:

```
python3 -m venv venv
```

Активируйте его (в начале строки терминала появится `(venv)`):

```
source venv/bin/activate
```

Установите библиотеку:

```
pip install nltk
```

> Если хотите пропустить виртуальное окружение, попробуйте сразу `pip install nltk` или `pip3 install nltk`. Если не сработает — вернитесь к шагам выше.

---

### Шаг 3. Перейдите в папку проекта в терминале

Самый простой способ получить путь к папке:

**macOS** — откройте терминал, напишите `cd ` (с пробелом) и перетащите папку проекта прямо в окно терминала. Путь подставится автоматически. Нажмите `Enter`.

**Windows** — откройте папку проекта в проводнике, нажмите правой кнопкой по пустому месту и выберите `Открыть в терминале`.

Проверьте что вы в нужной папке — должны видеть файл `distribution_of_email.py`:

```
ls
```

---

### Шаг 4. Положите письма в папку `inbox`

Папка `inbox` должна находиться внутри папки проекта рядом с файлом `distribution_of_email.py`:

```text
HAKATON-main/
├── distribution_of_email.py
├── run.sh
├── inbox/         ← положите письма сюда
└── ...
```

---

### Шаг 5. Запустите программу

```
chmod +x run.sh
./run.sh
```

Если `run.sh` не работает, запустите напрямую:

```
python3 distribution_of_email.py .
```

---

## Результат работы

После запуска в терминале появится:

```text
Распределение писем:

  mail_0001.txt                  Важное
  mail_0002.txt                  HR
  mail_0003.txt                  Спам

Папки в ./processed:

    3  Спам/
    2  Важное/
    1  HR/

Лог сохранён: ./processed/log.txt
```

Все письма будут разложены по папкам внутри `processed`.

> Письма **перемещаются**, а не копируются. После запуска папка `inbox` станет пустой.

## Тестовые данные

В папке `tests` лежат примеры файлов для проверки разных сценариев. Названия файлов подсказывают ожидаемую категорию:

```text
test_Счета.txt
test_HR.txt
test_Ошибки_binary.bin
test_НеРаспределено_nokeywords.txt
```

Чтобы прогнать тесты, скопируйте файлы из `tests` в папку `inbox` и запустите:

```
cp tests/* inbox/
./run.sh
```

## Формат письма

Для корректной обработки письмо должно содержать отправителя и получателя:

```text
From: employee@example.com
To: support@example.com
Subject: Не открывается GitLab
Date: 2026-06-01

Добрый день. Не могу войти в GitLab, нужен доступ к аккаунту.
```

Поддерживаются русские и транслитерированные заголовки: `From` / `От кого` / `Ot kogo`, `To` / `Кому` / `Komu`.
